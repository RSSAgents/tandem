В итоге в медиазапросе (@media (max-width: 1170px)) были добавлены правила, которые применяются только когда на body есть класс .ai-agent-page:
Сброшены отступы AppShell:
:global(body.ai-agent-page .mantine-AppShell-root) { --app-shell-navbar-offset: 0rem !important; --app-shell-navbar-width: 0rem !important; }
Спрятан сам navbar:
:global(body.ai-agent-page .mantine-AppShell-navbar) { display: none !important; }
Показан бургер:
:global(body.ai-agent-page .mantine-Burger-root) { display: flex !important; align-items: center; }

Почему это работает (и почему так безопасно) (объяснение от ИИ)
Mantine AppShell управляет размещением через CSS‑переменные (offset/width) — header и main получают padding/margin, рассчитываемые из этих переменных.
Если оставить переменную прежней, то даже скрытый navbar будет резервировать место слева. Меняя переменные, мы убираем резервирование корректно, без попыток «перекрывать» inline‑стили.
Burger может скрываться через Mantine в зависимости от breakpoint (иногда inline‑style). Принудительное display: flex !important показывает его там, где нужно.
Всё применено под селектором body.ai-agent-page, поэтому остальные страницы не пострадали.
